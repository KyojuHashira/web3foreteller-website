Frequently asked questions

Question: How does Web3 ForeTeller actually work?
Answer: Web3 ForeTeller generates a random "salt" based on events from the Web2 and Web3 worlds, then combines it for the best result. The final step adds some magic and gives users the answer they need!
Question: What is the difference between "Altruistic Answer" and "Burn some gas!" actions?
Answer: Clicking on the "Altruistic Answer" button calls the getRandomNumber() function, which returns a random value free of charge by fetching a random salt from the Web3 and Web2 worlds. This method works well but is not as powerful as the 'Burn some gas!'. Clicking on the "Burn some gas!" button triggers calls to the getRandomNumberWithGas() and getRandomSaltFromGasBurn() functions, which generate more web3 salt. This interaction allows the Web3 ForeTeller to gain better insight into your aura and provide a more accurate response! "Burn some Gas!" fee ~ $0.05.
Question: When did Web3 Foreteller come into existence and start?
Answer: Web3 Foreteller had its first local launch (with Denwer + Apache) on July 28th, 2023. It started serving people from the moment the Base Mainnet GA was launched. However, for the most curious, the backend was available from the beginning. Web3 ForeTeller will continue to serve people until it deems that its help is no longer needed.
Question: Will the project be further developed, or is it a finished project?
Answer: The randomization mechanisms will become more powerful and strive to absorb more and more salt from the Web2 and Web3 worlds to produce more accurate and better results. If new features need to be added, please create a vote on Snapshot. Any member with a Gitcoin passport above 15 points can create a vote.
Question: Is there a token planned for the project?
Answer: No token is planned at the moment, but maybe one day in the future.
Question: Will other networks be added?
Answer: At the moment, Web3 Foreteller works exclusively on Base. If you want the project to work on a new network, please create a vote on snapshot.org, and if it is demanded, the new network will be implemented.
